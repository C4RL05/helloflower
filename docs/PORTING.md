## Serialization format

```
helloflower1.3#name_<NAME>#corolla_<C>#corolla_<C>...      (sep '#', assign '_')
corolla: <x y z x y z ...>&colors=<r g b r g b>&petals=<n>&open=&spin=&sway=&curve=
         (sep '&', assign '=', element ' ')
```
- Control point: x = length along petal, y = curl, **z = half-width** (so tip z=0 → pointed tip).
- Legacy `helloflower1.0` header (all built-ins + Documents presets) applies
  `colorMult = 2 / textureMultiplier`. We write back as `1.3` (colorMult = 1).
- Write floats with a locale-invariant formatter (JS `String(n)` is fine).

## Extracting values from the binary scene

`HelloFlower-Unity/Assets/Scenes/HelloFlower.unity` is a **binary SerializedFile v9**
(Unity 3.5.1f2). Header (big-endian u32): metadataSize, fileSize, version, dataOffset.
For this scene `dataOffset = 63264` — object DATA starts there; type-tree metadata is before it.

Method used (see git history / scripts):
- **Flower multipliers**: scanned the data section for 4 consecutive plausible float32
  preceded by the Flower component's PPtr block. Identified Flower at offset 96616 by its
  signature: `stem`(local) `material`(pathID 2100000=Material) + 4 textures(pathID
  2800000=Texture2D, order alpha/lightmap/noise/shadow) `cameraTarget`(local). Result:
  `colorMultiplier 1.5, textureMultiplier 1.1, specularMultiplier 0.5, shadowMultiplier 0.7`.
- **RenderSettings ambient**: first object in the data section. Layout: m_Fog, FogColor
  (0.5,0.5,0.5), FogMode 3, density 0.01, linear 0–300, then `m_AmbientLight = 0.31373`
  (gray 80/255) at ~offset 63332. (Petal shader has `Fog{Mode Off}` so scene fog is irrelevant.)
- **Not recovered**: `DeviceAssets.petalCurvePointCount/petalSpanPointCount` (the DeviceAssets
  MonoBehaviour layout — bools + m_Name string — didn't yield a clean signature). Currently
  placeholders 32/12 in `FlowerConfig`; affects mesh density only.

## Petal painting

The painted per-corolla texture (32×16) is generated by `PetalPainter` from 4 mask arrays
decoded from the original PSDs (`scripts/extract-petal-channels.mjs` →
`petalChannels.generated.ts`). The PSDs are RGB grayscale; Unity derived alpha "from
grayscale", so the masks are the grayscale values.

```
shadowAlpha = clamp01((main.gray + sec.gray) * 0.5 * shadowMultiplier)   // per-corolla scalar
shadow      = lerp(shadowAlphas[i], 1, shadowAlpha)
pixel       = lerp(main*noise[i], secondary, color[i]) * lightmap[i] * textureMultiplier * shadow
alpha       = lightmap[i]          // packed as the specular mask (NOT opacity)
```

**Lightmap orientation bug (fixed):** the decoded lightmap was base-bright/tip-dark AND
opposed the base-dark shadow mask, so they cancelled into a flat petal. `loadPetalMasks`
now flips the lightmap vertically → tip-bright/base-dark, which reinforces the shadow and
restores the visible bright-tip → dark-base gradient. (Verify: `EL=12 UNLIT=1 node
scripts/capture-rose.mjs`.)

## Petal shader

Original `Flower/Petal 2.1`: `Lighting On; SeparateSpecular On; Material Diffuse(1,1,1,1)
Ambient(1,1,1,1) Specular[_SpecColor] Shininess[_Shininess≈0.1]; SetTexture { texture * primary }`.
`CameraBehaviour` drove two lights: a dominant camera "headlight" (~0.85) + weak main
directional (~0.6), plus the RenderSettings ambient.

`scene/PetalMaterial.ts` reproduces this as a custom ShaderMaterial:
```
primary = uAmbient + uCamColor·max(N·V,0) + uMainColor·max(N·mainDir,0)
final   = texture.rgb * primary + uSpecColor * pow(max(N·H,0), shininess) * texture.a
```
MeshPhong was wrong (three's PBR divides diffuse by π → far too dark). `petalEnv` holds the
shared uniforms (ambient 0.25, camColor 0.85, mainColor ~0.45, shininess 13). `updateSpecular`
ports `Corolla.UpdateSpecular`. Double-sided; normal flipped for back faces.

Color pipeline: `THREE.ColorManagement.enabled = false`, `outputColorSpace = LinearSRGB`
(gamma passthrough, matching the 2011 non-PBR look).

## Procedural color-picker gradient

`ui/gradientTexture.ts` reproduces the hand-tuned 77×256 `GradientSlider.png` (now a
test fixture) as a separable gradient: 33 rainbow hue stops down Y (HSV-angle interpolated,
full saturation) + a measured luma curve across X (brightness 0.254→1 left, whiten 0→0.867
right). ~99.6% match (mean ~1.1/255), verified by `tests/e2e/gradient.spec.ts` and
`scripts/{analyze,compare,save}-gradient.mjs`.

## Intro screen

main.ts: black background, top-down spinning rose, the `helloflower` logo
(`public/images/helloflower.png`, white-on-black) shown with `mix-blend-mode: screen`
(drops the black). Click anywhere → `endIntro`: UI fades in, background switches to the
light editing gradient, camera glides from top-down to the 3/4 view. A shared-flower URL
skips the intro. The overlay must NOT have a `z-index` (a stacking context would isolate
the blend from the canvas behind it).
